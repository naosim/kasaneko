# 2Dアクションゲーム『KASANEKO（傘猫）』要件定義書

## 1. プロジェクト概要
*   **プロジェクト名:** KASANEKO
*   **コンセプト:** 傘を巧みに操る猫が、キャンディを求めて冒険するレトロスタイルの2Dプラットフォーマー。
*   **ターゲット:** ブラウザで手軽に良質なアクションを楽しみたい層、レトロゲームファン。
*   **公開形態:** GitHub PagesによるWeb公開（ブラウザゲーム）。

---

## 2. 技術スタック
*   **開発エンジン:** [KAPLAY.js](https://kaplayjs.com/) (JavaScript/TypeScript用ゲームライブラリ)
*   **開発言語:** TypeScript
*   **ビルド・デプロイ:** Vite + GitHub Actions
*   **保存機能:** ブラウザの LocalStorage を利用した進行度保存

---

## 3. ゲームメカニクス
### 3.1 プレイヤー操作（傘アクション）
本作の核心となる、傘の状態管理システム。
*   **通常時:** 傘を閉じた状態。移動速度が速く、通常のジャンプが可能。
*   **ジャンプ中アクション:**
    *   空中でもう一度ジャンプボタンを押すと「傘が開く」。
    *   傘が開いている時にもう一度押すと「傘が閉じる」。
*   **傘・展開時の特性:**
    *   重力の影響が軽減され、ふんわりとした滞空が可能。
    *   横移動の慣性が維持されやすくなる。
    *   最大落下速度（終端速度）が制限される。

### 3.2 落下ダメージシステム
*   **判定基準:** 着地した瞬間の「落下速度（y軸速度）」が閾値を超えているか。
*   **ルール:** 
    *   高所から傘を閉じたまま落下し、一定速度を超えて着地するとダメージ（ライフ減少）。
    *   着地直前に傘を開くことで減速し、ダメージを回避できる。

### 3.3 クリア条件
*   **ゴールアイテム:** ステージの最後に配置された「キャンディ」。
*   **クリア演出:** キャンディ接触後、操作を無効化し、クリア専用アニメーションとファンファーレを再生。次ステージへ遷移する。

---

## 4. ユーザーインターフェース (UI)・操作体系
### 4.1 マルチデバイス対応
*   **キーボード:** 矢印キー（移動）、Zキー（ジャンプ/傘トグル）。
*   **コントローラー:** Gamepad API対応（A/Bボタンでの操作）。
*   **モバイル対応:** 画面上に独自の仮想パッドをオーバーレイ表示。
    *   左側：移動用ジョイスティック（または左右ボタン）。
    *   右側：ジャンプ/傘トグルボタン。

### 4.2 ビジュアルスタイル
*   **グラフィック:** 8bit/16bit風のドット絵。
*   **解像度:** レトロ感を出すための低解像度スケーリング設定。
*   **演出:** 傘の開閉アニメーション、ダメージ時の点滅、着地時の土煙エフェクト。

---

## 5. ステージ設計
*   **タイルマップ:** KAPLAYの記号記述式によるレベルデザイン。
*   **ギミック案:** 
    *   傘の浮遊特性を活かした「上昇気流」。
    *   傘を開いていると押し流される「強風」。
    *   落下ダメージのリスクを伴う「垂直構造のマップ」。

---

## 6. 非機能要件
*   **レスポンシブ対応:** 画面サイズに合わせてCanvasを最適にスケーリング。
*   **パフォーマンス:** 低スペックPCやスマートフォンでも動作する軽量なアセット設計。
*   **デプロイ:** GitHubレポジトリへのPushをトリガーに自動で最新版をPagesへ反映。
